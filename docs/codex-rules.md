# Codex Rules

- Use `docs/ai-website-roadmap-for-codex.md` as the source of truth for future sprint cycles
- Prefer simple reusable components
- Preserve the content schema
- Keep content editable via files
- Avoid unnecessary dependencies
- Validate mobile layouts
- Preserve accessible semantics
- Do not add decorative complexity without purpose
- Every major page needs a clear CTA
- Use the site-wide CTA hierarchy from the roadmap
- Primary CTA: `Start a conversation`
- Secondary CTA: `View case studies`
- Tertiary CTA: `See how I help`
- Complete work in roadmap-aligned sprints
- Validate each sprint before calling it complete
- Each agent must evaluate its own output at the end of its sprint
